ISSUE #{{ARG1}} を実装してください。

以下の手順で実行してください：

1. ISSUEの詳細を確認
   gh issue view {{ARG1}} でISSUEの内容を確認

2. 新しいブランチを作成
   git checkout -b fix/issue-{{ARG1}}

3. 実装を行う
   - 要件を正確に理解し、実装方針を決定
   - 必要なファイルを特定し、変更を実施
   - エラーハンドリングを適切に実装
   - 可能な限りテストを追加
   - 必要に応じてドキュメントを更新

4. 実装原則：
   - 既存のコードスタイルに従う
   - DRY原則を守る
   - SOLID原則を意識する
   - セキュリティを最優先に考慮
   - パフォーマンスへの影響を最小限に

5. 変更をコミット
   適切なコミットメッセージでコミット（例: "fix: implement #{{ARG1}} - [ISSUEタイトル]"）

6. プルリクエストを作成
   git push -u origin fix/issue-{{ARG1}}
   gh pr create --title "fix: implement #{{ARG1}}" --body "Closes #{{ARG1}}" --base main

実装が完了したら、mainブランチに戻ってください：
git checkout main

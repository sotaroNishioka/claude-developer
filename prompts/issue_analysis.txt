このプロジェクトを分析して、以下の観点から改善提案をしてください。

分析観点：
1. コードの品質向上（重複コード、複雑性の高い関数など）
2. セキュリティの改善（脆弱性、認証・認可の問題など）
3. パフォーマンスの最適化（非効率な処理、N+1問題など）
4. テストの充実（カバレッジ不足、重要な機能のテスト欠如など）
5. ドキュメントの改善（READMEの不備、コメント不足など）

JSON形式で以下の構造で出力してください：
{
  "issues": [
    {
      "title": "簡潔で明確なタイトル",
      "body": "詳細な説明と改善案",
      "priority": "high/medium/low",
      "type": "bug/feature/improvement/documentation"
    }
  ]
}

最大3つの重要な改善点を提案してください。
